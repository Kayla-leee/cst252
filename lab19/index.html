Again, class on Monday, November 16 was super helpful, as Wes re-made the code simple enough for us to finally understand. We weren’t sure how to go about that, since Wes didn’t teach us to call two APIs in one button. Our thought process was that now that we knew how to properly request one API, the way the second API is coded should be the same. We copied and pasted the code from the first $.ajax API request, changed the URL, variables, names, etc., and placed them in various places in the JS code. We tried placing it in a new .done(function(data), new $.ajax ​after ​the .done(function(data) from the first API, and other areas. We did some research, but struggled to get results due to not knowing the correct keywords. It took us a few searches to get to where we wanted, until we came across a code by Stack Overflow user Timothy Aaron, and he suggested nesting the second API request in the output function from the first API request. We didn’t want to scrap the code we already had, so we applied it differently to our code.
